## 1 整机简述

### 1.1 主要模块

SDK 处于中间层软件。SDK 层上接应用层和算法层，下接底层驱动层，用于衔接双方并驱动机器人正常运行。

### 1.2 SDK 与其他模块

#### SDK 层与底层驱动

电源板模块：

- 电源板与电池 BMS 连接，通过电源板这一层转接板接收电池数据。
- 当 SDK 打印出现 `Successfully opened serial port: /dev/xbot_battery`，表示已成功与电源板建立连接。
- 如果未出现该日志，需要排查电源板问题，常见原因是固件版本问题或接线缺失问题。

关节电机模块：

- 不同机型存在不同大小和样式的关节电机。
- 需要特别区分是否带抱闸。目前主要为 `B4 下肢三关节` 和 `M7.1 腰关节` 为带抱闸关节。
- 单个关节电机会通过线束线性串联为肢体关节，再通过耦合器以树形方式集合成一体，最终连接到 NUC 网口。

#### SDK 层与应用层/算法层

- 主要涉及 `XOS`、`MPC`、`RL`、`retarget` 等模块。
- 模块之间通过 ROS2 `Topic` / `Service` / `Action` 进行数据交互。
- `ros2_dev` docker 容器中包含多种脚本，用于快速执行相关程序。

## 2 操作手册使用说明

操作手册链接：ROBOTERA 机器人操作手册 2.0

### 2.1 什么情况下使用操作手册

1. 初次接触机器时，按手册从前向后顺序执行。
2. 后期开发、维护操作过程中，通过手册侧边目录快速定位需要使用的功能。

### 2.2 操作手册中包含什么

1. 机器人的启动方法：自动启动、手动启动，大部分已脚本化。
2. 后期维护：LOG 解析、整机标零、关节名称定义、固件升级、相关脚本简述等。

### 2.3 使用方法

#### 本地仿真使用

- 参考：本地 SDK 部署及注意事项。
- 特别注意：建议修改 `ROS_DOMAIN_ID`，防止与真机冲突。

主要修改位置：

- docker 中：`vim /root/.bashrc`，修改或添加 `export ROS_DOMAIN_ID=对应ID`
- 开机自启动设置中：`vim /root/xbot/hardware/supervisor_config/supervisord.conf`

ROS_DOMAIN_ID 建议范围：

| 部门 | ID 建议使用范围 |
| --- | --- |
| 算法 | 10 - 59 |
| 软件 | 60 - 109 |
| 测试 | 110 - 159 |
| 演示 | 110 - 159 |
| 售后 | 180 - 199 |

#### 全新下线机器

新下线机器的关节模组参数通常为空或默认值，需要依次进行：

1. 设置关节模组名称。
2. 选做：关节模组升级，查看关节版本号，确认是否为需求的模组固件软件版本。
3. 设置关节模组 PID。
4. 关节模组标零。
5. 关节方向一致性验证：配合 rviz 手动拉动关节确认。

#### 正常运行与后期维护机器

- 查看 log 日志，根据操作文档或开发解答进行操作。
- 修改配置文件时需配合排查，可在 docker 容器中修改相关参数。

常见配置目录/文件说明：

- `battery`：电池配置参数，可设置 log 打印频率和开启更多信息打印。
  - `battery_info_print_interval: 600.0`：电池信息打印间隔，单位秒。
  - `developer_mode: false`：开发者模式，启用后会打印更多调试信息。
- `controller`：controller 包含的关节信息设置。可以屏蔽关节；有关节被拔掉时，需要在此处屏蔽。屏蔽方法是在关节前加 `#`。
- `fault_manager`：设置错误码以及关节报错等级。
- `motion_manager`：机器人各状态下的 controller 挂载设置。
- `scripts`：相关脚本存储位置。
- `system_monitor`：进程监控管理。
- `urdf`：各型号机器设置，可以屏蔽关节。
- `xbot_action`：关节动作设置，可以修改或增删固定动作。
- `xbot_tool`：单程序工具参数设置。

## 3 常见错误问题简述

SDK 2.0 日志位置：

- `/system_log/log/dynamic_launch/`：仅自启动。
- `/system_log/log/xbot_control/xbot.log`：自启和手启均可，但需要 `SDK >= 2.0.91`。

常见问题：

- 电源板串口异常：查看是否存在电源板串口打开失败相关日志；结合 `Successfully opened serial port: /dev/xbot_battery` 判断连接状态。
- EC 状态切换失败：可能是模组固件版本太低、之前固件升级失败、或模组没有正常写入名称。
- 关节阶跃：关节收到了 `>= 0.628 rad`，约 36 度的阶跃。
- ENC 报错：需要根据 ENC 二级报错确认具体情况，例如 encoder error code / motor encoder error code。
- 关节模组未上电：可检查急停遥控器状态，文档提示“急停遥控器 D 一下”。
